## Что такое CRUD?

**CRUD** — это четыре основные операции для работы с данными:
- **C**reate — создание новой записи
- **R**ead — чтение (просмотр) записей
- **U**pdate — обновление (редактирование) записи
- **D**elete — удаление записи

## Resource-контроллеры

Вместо создания контроллера с пустыми методами, используйте пресет `--resource`:

```bash [terminal]
php artisan make:controller CourseController --resource
```

Это создаст контроллер со всеми методами CRUD:

| Метод | Для чего |
|-------|----------|
| `index()` | показать список ВСЕ записей |
| `create()` | показать форму создания |
| `store(Request)` | обработать данные и сохранить |
| `show($id)` | показать одну запись |
| `edit($id)` | показать форму редактирования |
| `update(Request, $id)` | обработать данные и обновить |
| `destroy($id)` | удалить запись |

## Resource маршруты

Вместо создания 7 маршрутов вручную, используйте:

```php [routes/web.php]
Route::resource('courses', CourseController::class);
```

Это автоматически создаст все 7 маршрутов!

## Использование моделей в параметрах

Вместо получения ID и поиска модели:

```php [app/Http/Controllers/CourseController.php]
public function show(int $id)
{
    $course = Course::findOrFail($id);  // долго
    return view('courses.show', ['course' => $course]);
}
```

Делайте так (Laravel сделает это автоматически):

```php [app/Http/Controllers/CourseController.php]
public function show(Course $course)  // Laravel сам найдёт по ID
{
    return view('courses.show', ['course' => $course]);
}
```

::tip
Если маршрут `courses/{course}`, то параметр метода `$course` (типа `Course`) получит найденную модель или выбросит 404.
::

## Создание записи

Получите валидные данные и создайте:

```php [app/Http/Controllers/CourseController.php]
public function store(Request $request)
{
    $data = $request->validate([
        'title' => ['required', 'string'],
        'description' => ['nullable', 'string'],
        'price' => ['required', 'numeric']
    ]);

    Course::create($data);  // создаёт новую запись

    return redirect()
        ->route('courses.index')
        ->with('message', 'Курс успешно создан!');
}
```

Форма создания:

```html [resources/views/courses/create.blade.php]
<form action="{{ route('courses.store') }}" method="POST">
    @csrf

    <input type="text" name="title" placeholder="Название">
    <textarea name="description" placeholder="Описание"></textarea>
    <input type="number" name="price" placeholder="Цена">

    <button type="submit">Создать</button>
</form>
```

## Обновление записи

```php [app/Http/Controllers/CourseController.php]
public function update(Request $request, Course $course)
{
    $data = $request->validate([
        'title' => ['required', 'string'],
        'description' => ['nullable', 'string'],
        'price' => ['required', 'numeric']
    ]);

    $course->update($data);  // обновляет существующую запись

    return redirect()
        ->route('courses.index')
        ->with('message', 'Курс успешно обновлен!');
}
```

Форма редактирования:

```html [resources/views/courses/edit.blade.php]
<form action="{{ route('courses.update', $course->id) }}" method="POST">
    @csrf
    @method('put')  <!-- важно! POST не может быть PUT -->

    <input type="text" name="title" value="{{ old('title', $course->title) }}">
    <textarea name="description">{{ old('description', $course->description) }}</textarea>
    <input type="number" name="price" value="{{ old('price', $course->price) }}">

    <button type="submit">Обновить</button>
</form>
```

::warning
Формы HTML могут отправлять только GET и POST. Для PUT/PATCH используйте `@method('put')`.
::

## Удаление записи

```php [app/Http/Controllers/CourseController.php]
public function destroy(Course $course)
{
    $course->delete();  // удаляет запись

    return redirect()
        ->route('courses.index')
        ->with('message', 'Курс успешно удалён!');
}
```

Форма удаления:

```html [resources/views/courses/index.blade.php]
@foreach ($courses as $course)
    <p>{{ $course->title }}</p>

    <form 
        action="{{ route('courses.destroy', $course->id) }}" 
        method="POST"
        style="display:inline;"
    >
        @csrf
        @method('delete')

        <button type="submit" onclick="return confirm('Вы уверены?')">
            Удалить
        </button>
    </form>
@endforeach
```

## Layout (главный шаблон)

Создайте общий шаблон для всех страниц:

```html [resources/views/layout.blade.php]
<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>@yield('title', 'Laravel App')</title>
    
    <link rel="stylesheet" href="{{ asset('css/bootstrap.min.css') }}">
</head>
<body>
    <header>
        <nav>
            <a href="{{ route('courses.index') }}">Курсы</a>
            @auth
                <span>{{ Auth::user()->name }}</span>
                <a href="{{ route('logout') }}">Выход</a>
            @endauth
            @guest
                <a href="{{ route('showLoginForm') }}">Вход</a>
            @endguest
        </nav>
    </header>

    <main class="container mt-4">
        @if (session()->has('message'))
            <div class="alert alert-success">
                {{ session('message') }}
            </div>
        @endif

        @yield('content')
    </main>

    <footer>
        <p>&copy; 2026 Laravel Guides</p>
    </footer>

    <script src="{{ asset('js/bootstrap.min.js') }}"></script>
</body>
</html>
```

Теперь в других шаблонах:

```html [resources/views/courses/index.blade.php]
@extends('layout')

@section('content')
    <h1>Курсы</h1>

    <a href="{{ route('courses.create') }}" class="btn btn-primary">
        Создать новый курс
    </a>

    @forelse ($courses as $course)
        <div class="card">
            <div class="card-body">
                <h5>{{ $course->title }}</h5>
                <p>{{ $course->description }}</p>
                <p class="price">{{ $course->price }} ₽</p>

                <a href="{{ route('courses.show', $course->id) }}" class="btn btn-sm btn-info">
                    Просмотр
                </a>
                <a href="{{ route('courses.edit', $course->id) }}" class="btn btn-sm btn-warning">
                    Редактирование
                </a>

                <form action="{{ route('courses.destroy', $course->id) }}" method="POST" style="display:inline;">
                    @csrf
                    @method('delete')
                    <button type="submit" class="btn btn-sm btn-danger" onclick="return confirm('Удалить?')">
                        Удалить
                    </button>
                </form>
            </div>
        </div>
    @empty
        <p>Курсов не найдено</p>
    @endforelse
@endsection
```

## Bootstrap для стилизации

**Bootstrap** — это CSS-библиотека, которая даёт вам готовые стили.

1. Скачайте файлы Bootstrap с официального сайта
2. Положите их в `public/css/` и `public/js/`
3. Подключите в `layout.blade.php`:

```html
<link rel="stylesheet" href="{{ asset('css/bootstrap.min.css') }}">
<script src="{{ asset('js/bootstrap.min.js') }}"></script>
```

Теперь используйте CSS-классы Bootstrap:

```html
<button class="btn btn-primary">Создать</button>
<button class="btn btn-danger">Удалить</button>

<div class="alert alert-success">Успех!</div>
<div class="alert alert-error">Ошибка!</div>

<div class="card">
    <div class="card-body">Содержимое карточки</div>
</div>

<div class="container">Контейнер со средней шириной</div>
```

::tip
Используйте Zeal или документацию Bootstrap, чтобы копировать готовые компоненты (формы, кнопки, таблицы и т.д.) вместо создания с нуля.
::

## Практическая работа

1. Создайте resource-контроллеры:
   ```bash
   php artisan make:controller CourseController --resource
   php artisan make:controller ApplicationController --resource
   ```

2. Добавьте resource-маршруты:
   ```php
   Route::resource('courses', CourseController::class);
   Route::resource('applications', ApplicationController::class);
   ```

3. Создайте представления для каждого метода (index, create, edit, show)

4. Реализуйте все CRUD-методы в контроллерах

5. Добавьте Bootstrap для красивого оформления

6. Проверьте, что все операции работают:
   - Создание нового курса
   - Просмотр списка курсов
   - Редактирование курса
   - Удаление курса

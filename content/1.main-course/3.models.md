## Что такое модели?

**Модели** (или модели Eloquent) — это классы, которые представляют таблицы вашей БД. Одна таблица = одна модель.

Вместо того, чтобы писать SQL-запросы напрямую, вы работаете с моделями на PHP — это проще и безопаснее.

## Создание модели

Создайте новую модель командой:

```bash [terminal]
php artisan make:model Course
```

Будет создан файл `app/Models/Course.php`.

::tip
Названия моделей должны быть:
- В **единственном числе** (таблица `courses` → модель `Course`)
- В **PascalCase** (если составное название, например `user_roles` → модель `UserRole`)
::

## Структура модели

Вот пример простой модели:

```php [app/Models/Course.php]
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class Course extends Model
{
    protected $fillable = [
        'title',
        'wanted_start_date',
        'payment_method'
    ];
}
```

### `$fillable` — какие поля можно установить?

Это массив полей, которые **разрешено** автоматически заполнять при создании или изменении записи.

::warning
Если поле не в `$fillable`, оно не будет сохранено в БД, даже если вы его передали!
::

Не нужно добавлять в `$fillable`:
- `id` (добавляется автоматически)
- `created_at` и `updated_at` (добавляются автоматически)

## Использование моделей

### Получение данных из БД

```php [app/Http/Controllers/CourseController.php]
<?php

use App\Models\Course;

class CourseController
{
    public function index()
    {
        $courses = Course::all();  // получить ВСЕ курсы
        dd($courses);              // вывести для проверки
    }
}
```

Основные методы получения данных:
- `Course::all()` — получить все записи
- `Course::find(1)` — получить запись с ID = 1
- `Course::findOrFail(1)` — получить или выбросить 404
- `Course::where('title', 'Laravel')->first()` — найти первую с условием

### Collection (коллекция) vs Query (запрос)

Это важно понимать:

**Collection** — готовые данные (можно использовать сразу):
```php
$courses = Course::all();           // Collection
$course = Course::find(1);          // модель Course
$course->title;                     // готово использовать
```

**Query** — запрос, ещё не выполнен (нужно дополнить):
```php
$query = Course::where('id', '>', 5);  // Query Builder
// Можем добавить ещё условия
$query->where('active', true);
$result = $query->get();             // теперь выполняется
```

::note
Если метод заканчивается на `()` и это не `all()` или `find()`, скорее всего это запрос. Для выполнения используйте `.get()` или `.first()`.
::

## Связи между моделями

Связи позволяют легко получать связанные данные.

### Связь 1:1 (один-к-одному)

Например, у каждого пользователя есть одна роль. В таблице `users` есть поле `user_role_id`, которое ссылается на `user_roles`.

В модели `User`:

```php [app/Models/User.php]
<?php

namespace App\Models;

use Illuminate\Foundation\Auth\User as Authenticatable;

class User extends Authenticatable
{
    public function role()
    {
        return $this->hasOne(UserRole::class, 'id', 'user_role_id');
    }
}
```

Теперь используем связь:

```php [app/Http/Controllers/UserController.php]
$user = User::find(1);
echo $user->role->title;  // выведет название роли этого пользователя
```

::warning
Если связь не работает (возвращает null), попробуйте поменять параметры местами:

```php
// Не работает - get null
return $this->hasOne(UserRole::class, 'user_role_id', 'id');

// Работает
return $this->hasOne(UserRole::class, 'id', 'user_role_id');
```
::

### Связь 1:М (один-ко-многим)

Например, одна роль может быть у многих пользователей.

В модели `UserRole`:

```php [app/Models/UserRole.php]
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;

class UserRole extends Model
{
    public function users()
    {
        return $this->hasMany(User::class, 'user_role_id', 'id');
    }
}
```

Используем:

```php
$adminRole = UserRole::where('title', 'admin')->first();
$allAdmins = $adminRole->users;  // все пользователи с ролью админа
```

## Быстрая проверка данных

Если нужно просто проверить, какие данные приходят из БД:

```php
$user = User::find(1);

// Показывает много служебной информации (сложно читать)
dd($user);

// Показывает только нужные данные (понятнее)
dd($user->toArray());
```

## Практическая работа

1. Создайте модели для всех таблиц (User, UserRole, Course, Application, ApplicationStatus)

2. В каждой модели добавьте `$fillable` с нужными полями

3. Добавьте связи между моделями согласно заданию

4. Создайте контроллер и методом выведите:
   - Список всех пользователей через `User::all()`
   - Через методы моделей выведите имя админа и его роль

## Что такое маршруты?

**Маршруты** (routes) — это правила, которые описывают, как обрабатывать запрос.

Когда пользователь открывает `http://localhost/login`, маршрутизатор Laravel должен понять: "ОК, это GET-запрос на `/login`, отправим его в контроллер AuthController, метод showLoginForm".

## Создание маршрутов

Все маршруты пишутся в файле `routes/web.php`:

```php [routes/web.php]
<?php

use Illuminate\Support\Facades\Route;
use App\Http\Controllers\CourseController;
use App\Http\Controllers\AuthController;

// Главная страница
Route::get('/', [CourseController::class, 'index']);

// Страница авторизации (показать форму)
Route::get('login', [AuthController::class, 'showLoginForm']);

// Обработка данных из формы авторизации
Route::post('login', [AuthController::class, 'login']);
```

### Разница между GET и POST:

- **GET** — просмотр страницы (введёте в адресную строку)
  ```
  http://localhost/login
  ```

- **POST** — отправка данных (когда нажимаете кнопку "Отправить" в форме)

## HTTP методы:
- `GET` — просмотр данных
- `POST` — создание/отправка новых данных
- `PUT`/`PATCH` — редактирование
- `DELETE` — удаление

## Динамические маршруты

Если нужно передать ID в URL:

```php [routes/web.php]
Route::get('courses/{id}', [CourseController::class, 'show']);
```

Тогда:
- `http://localhost/courses/1` → `id = 1`
- `http://localhost/courses/5` → `id = 5`

## Именование маршрутов

Каждому маршруту можно дать имя, чтобы позже на него ссылаться:

```php [routes/web.php]
Route::get('login', [AuthController::class, 'showLoginForm'])->name('showLoginForm');
Route::post('login', [AuthController::class, 'login'])->name('login');
```

Теперь в шаблоне можно использовать:

```html [resources/views/navbar.blade.php]
<a href="{{ route('showLoginForm') }}">Логин</a>
```

Дальше изменять URL будет очень просто — просто измените маршрут, и все ссылки обновятся автоматически!

## Контроллеры

**Контроллер** — это класс, который содержит методы для обработки запросов.

### Создание контроллера

```bash [terminal]
php artisan make:controller AuthController
```

Будет создан файл `app/Http/Controllers/AuthController.php`.

### Структура контроллера

```php [app/Http/Controllers/AuthController.php]
<?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;

class AuthController extends Controller
{
    // Метод для показа страницы авторизации
    public function showLoginForm(Request $request)
    {
        return view('login');  // возвращаем шаблон
    }

    // Метод для обработки данных из формы
    public function login(Request $request)
    {
        // Обработка данных
    }
}
```

## Получение данных из запроса

Используется объект `Request`:

```php [app/Http/Controllers/AuthController.php]
public function login(Request $request)
{
    // Получить значение из поля <input name="login">
    $login = $request->input('login');
    
    // Или короче
    $login = $request->login;
    
    // Получить все данные
    $data = $request->all();
    
    dd($login, $data);
}
```

## Получение параметров из маршрута

Параметры из `{id}` передаются как параметры метода:

```php [routes/web.php]
Route::get('courses/{id}', [CourseController::class, 'show']);
```

```php [app/Http/Controllers/CourseController.php]
public function show(Request $request, int $id)
{
    echo "Вы запросили курс: $id";
}
```

Несколько параметров:

```php [routes/web.php]
Route::get('users/{user}/courses/{course}', [CourseController::class, 'show']);
```

```php [app/Http/Controllers/CourseController.php]
public function show(Request $request, string $user, string $course)
{
    echo "Пользователь: $user, Курс: $course";
}
```

## Валидация данных

Валидация проверяет, что данные из формы правильные.

```php [app/Http/Controllers/AuthController.php]
public function login(Request $request)
{
    $data = $request->validate([
        'login' => ['required'],           // обязательное поле
        'password' => ['required', 'min:8'] // обязательное, минимум 8 символов
    ]);
    
    // Если валидация прошла, $data содержит чистые данные
    dd($data);
}
```

### Основные правила валидации:

| Правило | Описание |
|---------|---------|
| `required` | поле НЕ должно быть пустым |
| `nullable` | поле может быть пустым |
| `string` | обязательно строка |
| `email` | проверка формата email |
| `integer` | обязательно число |
| `min:8` | минимум 8 символов (или минимальное число) |
| `max:255` | максимум 255 символов |
| `unique:table,column` | значение уникально в БД |
| `exists:table,column` | значение существует в БД |
| `regex:/[A-Z]/` | соответствует регулярному выражению |

Пример со сложной валидацией:

```php [app/Http/Controllers/AuthController.php]
public function login(Request $request)
{
    $data = $request->validate([
        'password' => [
            'required',
            'min:8',
            'regex:/[A-Z]/',  // хотя бы одна заглавная
            'regex:/[a-z]/',  // хотя бы одна строчная
            'regex:/[0-9]/',  // хотя бы одна цифра
        ],
    ]);
}
```

::tip
После валидации `$request->validate()` выбросит ошибку, если данные не проходят проверку, и перенаправит пользователя обратно на форму. Если проверка прошла, метод возвращает массив с валидными данными.
::

## Практическая работа

1. Создайте контроллер: `php artisan make:controller AuthController`

2. Добавьте два метода:
   - `showLoginForm(Request $request)` — возвращает `view('login')`
   - `login(Request $request)` — валидирует данные (login и password)

3. Создайте маршруты:
   ```php
   Route::get('login', [AuthController::class, 'showLoginForm'])->name('showLoginForm');
   Route::post('login', [AuthController::class, 'login'])->name('login');
   ```

4. Протестируйте маршруты в браузере

## Что такое миграции?

**Миграции** — это файлы, которые описывают изменения в вашей базе данных (создание и удаление таблиц). Вместо того, чтобы создавать таблицы вручную в phpMyAdmin, мы пишем код на PHP.

Это очень удобно, так как:
- Вы можете легко пересоздать всю БД на другом компьютере просто запустив одну команду
- Все изменения хранятся в файлах проекта
- Легко отслеживать историю изменений

## Создание новой миграции

Для создания новой миграции используйте команду:

```bash [terminal]
php artisan make:migration create_applications_table
```

Обратите внимание на формат названия: **create_ТАБЛИЦА_table**. Это стандартное соглашение в Laravel.

После запуска команды создастся файл миграции в папке `database/migrations/`. 

## Структура миграции

Вот пример миграции с разными типами данных:

```php [database/migrations/create_users_table.php]
public function up(): void
{
    Schema::create('users', function (Blueprint $table) {
        $table->id();                                 // уникальный ID
        
        $table->string('name');                       // текст до 255 символов
        $table->text('description');                  // длинный текст до 65535 символов
        $table->integer('age');                       // число
        $table->string('email')->unique();            // уникальная строка
        $table->string('note')->nullable();           // может быть NULL (пусто)
        $table->boolean('is_admin')->default(true);   // true/false с значением по умолчанию
        $table->enum('status', ['new', 'accepted'])->default('new'); // выбор из списка
        
        $table->timestamps();                         // created_at и updated_at (время создания и изменения)
    });
}
```

### Основные типы данных:
- `id()` — уникальный номер записи
- `string('name')` — текст до 255 символов
- `text('description')` — длинный текст
- `integer('age')` — целые числа
- `boolean('active')` — да/нет
- `enum('status', ['new', 'accepted'])` — выбор из списка
- `timestamp('date')` — дата и время
- `timestamps()` — автоматически добавляет created_at и updated_at

## Связь между таблицами

Когда одна таблица ссылается на другую, используются **вторичные ключи**.

Например, у нас есть таблицы `users` и `user_roles`. Каждый пользователь имеет роль, поэтому в таблице `users` нужно добавить поле, которое указывает на роль:

```php [database/migrations/create_users_table.php]
public function up(): void
{
    Schema::create('users', function (Blueprint $table) {
        $table->id();
        $table->string('name');
        
        $table->foreignId('user_role_id')  // вторичный ключ
            ->constrained('user_roles');   // ссылается на таблицу user_roles
        
        $table->timestamps();
    });
}
```

::tip
Если название составное, используйте **snake_case** — например, `user_role_id`, `application_status_id`.
::

## Порядок выполнения миграций

**Важно!** Некоторые миграции должны выполниться раньше других.

Например, таблица `user_roles` должна создаться **раньше**, чем `users`, потому что в `users` есть ссылка на `user_roles`.

Если вам нужно изменить порядок, **измените дату в названии файла миграции**:

- Вместо: `2026_03_20_085448_create_users_table.php`
- На: `2013_03_20_085448_create_users_table.php`

Тогда эта миграция выполнится раньше.

## Запуск миграций

После создания миграций запустите их командой:

```bash [terminal]
php artisan migrate
```

Эта команда создаст все таблицы в вашей БД.

### Другие полезные команды:

```bash [terminal]
php artisan migrate:reset      # откатить все миграции (удалить таблицы)
php artisan migrate:refresh    # пересоздать БД (откатить и заново создать)
php artisan migrate --seed     # создать таблицы И заполнить их тестовыми данными
```

## Сидеры (заполнение БД тестовыми данными)

**Сидеры** — это файлы, которые добавляют тестовые данные в БД. Главный файл сидера — это `DatabaseSeeder`:

```php [database/seeders/DatabaseSeeder.php]
<?php

namespace Database\Seeders;

use Illuminate\Support\Facades\DB;
use Illuminate\Database\Seeder;

class DatabaseSeeder extends Seeder
{
    public function run(): void
    {
        // Добавляем две роли
        DB::table('user_roles')->insert([
            'title' => 'admin'
        ]);
        
        DB::table('user_roles')->insert([
            'title' => 'client'
        ]);

        // Найдем ID админской роли
        $adminRoleId = DB::table('user_roles')
            ->where('title', 'admin')
            ->value('id');

        // Добавляем админа
        DB::table('users')->insert([
            'user_role_id' => $adminRoleId,
            'login' => 'Admin',
            'password' => 'KorokNET',
        ]);
    }
}
```

Запустите сидер:

```bash [terminal]
php artisan db:seed
```

::note
Если вы хотите пересоздать всё с нуля и сразу заполнить тестовыми данными, используйте:
```bash
php artisan migrate:refresh --seed
```
::

## Практическая работа

1. Создайте миграции для следующих таблиц (в правильном порядке):
   - `user_roles` (должна быть первой)
   - `users`
   - `courses`
   - `application_statuses`
   - `applications`

2. Заполните каждую миграцию нужными полями (смотрите задание в `TASK.md`)

3. Запустите миграции: `php artisan migrate`

4. Заполните `DatabaseSeeder` данными по заданию

5. Запустите сидер: `php artisan migrate:refresh --seed`

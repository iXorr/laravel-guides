## Общая информация

**Laravel** — это целая экосистема для разработки веб-приложений.

В рамках дем. экзамена нам не нужен весь функционал, поэтому мы сосредоточимся только на базовых возможностях. Все практические задания этого курса связаны между собой — в итоге вы реализуете полноценный экзаменационный вариант.

Будет использоваться версия ``10.x``.

::

Laravel использует паттерн **MVC**:

* **Models** — специальные классы, которые представляют таблицы из БД; их фишка в том, что вы можете создавать, просматривать, изменять и удалять записи из таблиц, *не используя SQL-запросы*
* **Controllers** — приём HTTP-запросов (например, POST-запрос на регистрацию) и выдача ответов (например, сообщение об успехе или ошибке)
* **Views** — отображение (*Blade*-шаблоны)

::note
**Blade** — шаблонизатор с особым синтаксисом. Он завязан на *двойных фигурных скобках*, внутри которых вычисляемое значение — php-переменные, и *@-директивах* (условный рендеринг, циклы).
::

### Основные папки

* **app** — ядро всего приложения: модели, контроллеры
* **database** — создание таблиц в БД (``migrations``) и их наполнение (``seeders``) без *phpMyAdmin*
* **public** — публичные файлы, к которым есть доступ по прямой ссылке (например, ``http://localhost/favicon.ico``)
* **resources/views** — отображение страниц; в подпапке ``views`` хранятся специальные Blade-шаблоны
* **routes** — маршрутизация запросов; о ней — следующий пункт
* **.env** — файл окружения; в нём будем писать только название БД, пароль к ней и пользователя

## Маршрутизация

Если вы пишете сайт на чистом PHP, то обращаетесь буквально к скриптовым файлам: ``http://localhost/products.php?id=1``.

В условиях Laravel это невозможно по двум причинам:

- Сложная архитектура.
- Обращение к файлам напрямую небезопасно.

В Laravel мы обращаемся по таким URL: ``http://localhost/users/1``. Сервер, разбивая строку обращения, передаёт наш запрос контроллеру.

::note
Мы работаем ТОЛЬКО с web-запросами (т.е. они возвращают html-странички) и, соответственно, с файлом ``routes/web.php``.
::

Подробнее о машрутах будет рассказано в теме по контроллерам.

## Жизненный цикл запроса

Это — классическая схема MVC-приложения, но адаптированная под Laravel (добавлен узел с маршрутами).

![MVC цикл](/images/mvc.png)

Запрос пользователя проходит в таком порядке:

1. URL отлавливается маршрутизатором (файлом ``routes/web.php``).
2. Маршрут передаётся контроллеру.
3. Контроллер, если ему было что-то передано, сперва обращается к модели.
4. Модель возвращает данные из БД для контроллера.
5. Контроллер собирает данные для ответа и обращается к шаблону (view).
6. Шаблон, получив данные для ответа, пихает их в html-теги и возвращает обычную html-страницу.
7. Контроллер возвращает пользователю html-страницу.

## Утилита Artisan

В Laravel почти всё — это PHP-классы, и мы должны создавать новые классы с расчётом, что в них УЖЕ будут какие-то подключения (через ``use``), конструкторы, методы и прочее.

Используйте специальную утилиту — **artisan**.

Файл ``artisan`` есть в корне проекта. Мы обращаемся к нему, как к обычному PHP-скрипту через консоль:

```bash
php artisan make:controller ExampleController
```

Если команды PHP нет в консоли, пишите полный путь до интерпретатора PHP. Для «Open Server» это выглядит примерно так:

```bash
C:\OSPanel\modules\PHP\PHP_8.1\php.exe artisan make:controller ExampleController
```

*Команды artisan* мы ещё разберём отдельно.

## Фасады и Method Chaining

В Laravel есть сложные системы классов — например, *сервисы маршрутизации*, *связи с БД* или *аутентификации*. Но к этим системам можно легко обращаться через **фасады** — специальные классы, которые предоставляют статический интерфейс для разработчика.

Например, метод ``get()`` фасада **Route** позволяет использовать себя как статически (через двоеточие), так и с привязкой к объекту (через лямбда-выражение).

```php
Route::get('/', function() {
    return 'hello!';
});

// Использование того же get(),
// но в контексте уже созданного объекта

Route::middleware('auth')->get('/', function() {
    return 'hello!';
});
```

Это работает благодаря технике ООП — **Method chaining** — при которой несколько методов объекта вызываются последовательно в одной строке, а каждый из этих методов — возвращает текущий объект (через ``return $this``).

```php
$users = User::where->orderBy('name')->take(10)->get();
```

## Практическая часть

::steps{level="4"}

#### Проверка проекта

Убедитесь, что сайт работает.

#### Создание контроллера

Убедитесь, что вы можете пользоваться утилитой `artisan`.

Создайте через него контроллер ``DBCheckController``.

```bash
php artisan make:controller DBCheckController
```

#### Наполнение контроллера

В контроллер добавьте метод, который будет вызываться маршрутизатором.

```php
<?php

// app/Http/Controllers/DBCheckController.php

namespace App\Http\Controllers;

use Illuminate\Http\Request;
use Illuminate\Support\Facades\DB; // подключите этот фасад вручную

class DBCheckController extends Controller
{
    public function index(Request $request)
    {
        dd(DB::connection()->getPdo());
    }
}
```

#### Привязка к роутеру

Привяжите маршрут главной страницы к методу ``index`` контроллера **DBCheckController** — это передаётся как массив, где первый аргумент — класс контроллера, второй — метод внутри него.

```php
<?php

// routes/web.php

use Illuminate\Support\Facades\Route;

use App\Http\Controllers\DBCheckController; // сами подключите контроллер

Route::get('/', [DBCheckController::class, 'index']);
```

#### Вывод на странице

Перейдите на сайт. Через окно отладки должен выводится объект *PDO*.

```php
PDO {
  inTransaction: false
  attributes: {}
}
```

## Отладка

* `dump()` — просто вывод
* `dd()` — вывод + остановка выполнения

Аналог `var_dump()`, но удобнее.

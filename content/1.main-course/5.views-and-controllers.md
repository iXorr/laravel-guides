## Что такое представления (Views)?

**Представления** — это файлы с HTML-разметкой, которые смешивают статический HTML с динамическими данными из контроллера.

Все файлы представлений находятся в папке `resources/views/`.

### Простое представление

```html [resources/views/login.blade.php]
<!DOCTYPE html>
<html>
<head>
    <title>Авторизация</title>
</head>
<body>
    <form action="/login" method="POST">
        <input type="text" name="login" placeholder="Логин">
        <input type="password" name="password" placeholder="Пароль">
        <button type="submit">Войти</button>
    </form>
</body>
</html>
```

## Возврат представления из контроллера

```php [app/Http/Controllers/AuthController.php]
public function showLoginForm(Request $request)
{
    return view('login');  // возвращает resources/views/login.blade.php
}
```

::tip
Путь к файлу указывается без расширения `.blade.php`. Если файл вложенный (`resources/views/courses/index.blade.php`), используйте точку: `view('courses.index')`.
::

## Передача данных в представление

```php [app/Http/Controllers/CourseController.php]
public function index()
{
    $courses = Course::all();
    
    return view('courses.index', [
        'courses' => $courses,
        'title' => 'Все курсы'
    ]);
}
```

В представлении эти данные доступны как переменные:

```html [resources/views/courses/index.blade.php]
<h1>{{ $title }}</h1>

@foreach ($courses as $course)
    <p>{{ $course->title }}</p>
@endforeach
```

## Blade-директивы

**Blade** — это шаблонизатор Laravel. Он позволяет использовать специальные команды.

### Вывод значения

```html
{{ $variable }}           <!-- выведет значение -->
{{ $user->name }}         <!-- выведет свойство объекта -->
{!! $htmlContent !!}      <!-- выведет HTML без экранирования -->
```

### Условия

```html
@if ($condition)
    <p>Условие истинно</p>
@elseif ($other)
    <p>Другое условие</p>
@else
    <p>Ничего не истинно</p>
@endif
```

### Циклы

```html
@foreach ($items as $item)
    <p>{{ $item->name }}</p>
@endforeach

@forelse ($items as $item)
    <p>{{ $item->name }}</p>
@empty
    <p>Список пуст</p>
@endforelse
```

## Проверка аутентификации

```html
@auth
    <!-- Пользователь вошёл в систему -->
    <p>Добро пожаловать, {{ Auth::user()->name }}!</p>
@endauth

@guest
    <!-- Пользователь не вошёл -->
    <p><a href="{{ route('showLoginForm') }}">Войти</a></p>
@endguest
```

## Проверка роли (авторизация)

Сначала определите "гейт" в провайдере:

```php [app/Providers/AuthServiceProvider.php]
<?php

use App\Models\User;
use Illuminate\Support\Facades\Gate;

public function boot(): void
{
    Gate::define('admin', fn (User $user) 
        => $user->role->title === 'admin'
    );
}
```

Теперь в представлении:

```html
@can('admin')
    <!-- Видят только админы -->
    <p>Панель управления</p>
@endcan
```

## Редиректы

После выполнения действия обычно нужно перенаправить пользователя:

```php [app/Http/Controllers/AuthController.php]
public function login(Request $request)
{
    // ... обработка ...
    
    return redirect('/');  // на главную
}

// Или с названием маршрута
return redirect()->route('courses.index');

// Вернуться назад
return back();
```

## Флеш-сообщения

Флеш-сообщения — это сообщения, которые выводятся один раз после редиректа.

Добавление сообщения в контроллере:

```php [app/Http/Controllers/CourseController.php]
public function store(Request $request)
{
    // ... создание курса ...
    
    return redirect()
        ->route('courses.index')
        ->with('message', 'Курс успешно создан!');
}
```

Вывод в представлении:

```html [resources/views/courses/index.blade.php]
@if (session()->has('message'))
    <div class="alert">
        {{ session('message') }}
    </div>
@endif
```

## Сохранение введённых данных

Если валидация не прошла, нужно вернуть пользователя на форму с его старыми значениями:

```php [app/Http/Controllers/AuthController.php]
public function login(Request $request)
{
    $data = $request->validate([
        'login' => ['required'],
        'password' => ['required']
    ]);
    
    // Если ошибка
    return back()->withInput();
}
```

В форме восстановите значения:

```html [resources/views/login.blade.php]
<input 
    type="text" 
    name="login" 
    value="{{ old('login') }}"
>
```

## Отображение ошибок валидации

```html [resources/views/login.blade.php]
@if ($errors->any())
    <div class="alert alert-danger">
        <ul>
            @foreach ($errors->all() as $error)
                <li>{{ $error }}</li>
            @endforeach
        </ul>
    </div>
@endif

<!-- Или конкретная ошибка поля -->
@error('login')
    <span class="error">{{ $message }}</span>
@enderror

<input type="text" name="login">
```

## CSRF токены

Для POST-запросов **обязательно** добавляйте CSRF-токен:

```html [resources/views/login.blade.php]
<form method="POST" action="{{ route('login') }}">
    @csrf  <!-- Это важно! -->
    
    <input type="text" name="login">
    <input type="password" name="password">
    <button type="submit">Войти</button>
</form>
```

::warning
Без `@csrf` Laravel выбросит ошибку 419. Это защита от CSRF-атак.
::

## Использование PHP-кода в шаблоне

```html [resources/views/course.blade.php]
@php
    $price = 1000;
    $discount = 0.1;
    $finalPrice = $price - ($price * $discount);
@endphp

<p>Финальная цена: {{ $finalPrice }}</p>
```

## Практическая работа

1. Создайте папку `resources/views/`

2. Создайте файл `resources/views/login.blade.php` с формой авторизации

3. Добавьте `@csrf` в форму

4. В контроллере `AuthController`:
   - метод `showLoginForm()` возвращает `view('login')`
   - метод `login()` валидирует данные

5. Обработайте ошибки валидации в шаблоне с помощью `@error`

6. Добавьте флеш-сообщение после успешной авторизации
